# Project Memory

## Durable Intent

The user wants a YouTube content automation dashboard, not just a prompt chain. The system should eventually automate category selection, top video discovery, video/script analysis, fact-checking, research enrichment, script planning, storyboard creation, image and video generation prompts, media generation, voice/subtitles/BGM, video assembly, thumbnail/description generation, and YouTube publishing.

## Harness Preference

The project should follow a harness-first approach:

- scripts and checks for deterministic work
- skills for repeatable creative and analytical workflows
- clear agent roles for strategy, research, writing, media, QA, security, and publishing
- persistent docs instead of oversized one-off prompts
- approval gates for money, external publishing, copyright/policy risk, and factual uncertainty

## Open Questions

- Which YouTube data source will be used first: official YouTube Data API, a third-party trend API, or manual seed URLs?
- Which model providers will be used for LLM, image generation, video generation, TTS, ASR, and music?
- Which dashboard stack should be used for MVP: Next.js, Streamlit, or another local-first stack?
- Should the first MVP target Shorts, long-form videos, or both?
- What channel tone, audience, and forbidden topics should become default policy?
